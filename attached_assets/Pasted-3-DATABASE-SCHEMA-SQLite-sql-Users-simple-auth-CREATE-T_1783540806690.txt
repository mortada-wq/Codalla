6. IMPLEMENTING THE AI STREAMING & MODEL TOGGLING
The magic: route AI requests to any provider based on user’s chosen model.

Backend logic (/ai/chat):

javascript
// Pseudocode
app.post('/api/v1/ai/chat', async (req, res) => {
  const { conversationId, messages, modelName, projectId } = req.body;
  // 1. Lookup model config from DB (provider, baseURL, apiKey)
  const modelConfig = await getModelConfig(modelName);
  const apiKey = decrypt(modelConfig.api_key);
  const baseURL = modelConfig.base_url; // e.g. https://api.siliconflow.cn/v1

  // 2. Build request payload
  const systemMsg = buildSystemContext(projectId); // inject file tree, open files content (if token limit allows)
  const fullMessages = [systemMsg, ...messages];

  // 3. Set up SSE stream to client
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    Connection: 'keep-alive',
  });

  // 4. Use axios with stream
  const response = await axios({
    method: 'post',
    url: `${baseURL}/chat/completions`,
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json',
    },
    data: {
      model: modelConfig.model_id,
      messages: fullMessages,
      stream: true,
    },
    responseType: 'stream',
  });

  let totalCompletionTokens = 0;
  response.data.on('data', (chunk) => {
    // Parse SSE lines, forward to client, accumulate tokens from delta
    const lines = chunk.toString().split('\n').filter(line => line.startsWith('data: '));
    for (const line of lines) {
      const jsonStr = line.replace('data: ', '');
      if (jsonStr === '[DONE]') break;
      const parsed = JSON.parse(jsonStr);
      const content = parsed.choices[0]?.delta?.content || '';
      res.write(`data: ${JSON.stringify({ content })}\n\n`);
      // Some APIs return usage in final chunk – track it
      if (parsed.usage) {
        totalCompletionTokens += parsed.usage.completion_tokens;
        // Calculate cost and save to DB after stream ends
      }
    }
  });

  response.data.on('end', () => {
    res.write('data: [DONE]\n\n');
    res.end();
    // Save message and cost to DB
  });
});

7. GIT OPERATIONS WITH isomorphic-git
We'll clone repos directly into the Replit filesystem using isomorphic-git (lightweight, no native git dependency). For GitHub private repos, use the user’s GitHub token.

Clone function:

javascript
const git = require('isomorphic-git');
const http = require('isomorphic-git/http/node');
const fs = require('fs');

async function cloneRepo(projectDir, remoteUrl, branch, token) {
  await git.clone({
    fs,
    http,
    dir: projectDir,
    url: remoteUrl,
    ref: branch,
    singleBranch: true,
    depth: 1,
    onAuth: () => ({ username: token, password: 'x-oauth-basic' }),
  });
}
Other operations (pull, push, commit, status) similarly using isomorphic-git API.

GitHub API (octokit) for repo metadata, creating issues, etc.

8. CODE FIXING & DIFF WORKFLOW
User right-clicks on a file (or code selection) → “Fix with AI”.

Frontend captures the file path, content, and user’s optional instruction.

Request to /api/v1/ai/fix with:

json
{
  "file": "src/utils.js",
  "code": "whole file content",
  "instruction": "refactor to use async/await and fix the memory leak",
  "model": "DeepSeek V3"
}
Backend constructs a prompt:
System: You are an expert code fixer. Return ONLY the full corrected file content in a code block.
User: file: src/utils.js ``` ...``` instruction: ...

Response is parsed to extract new code.

Backend computes a diff (using diff library) and sends back both old and new content and the diff.

Frontend shows <DiffViewer> with accept/reject. On accept, file is written back.
